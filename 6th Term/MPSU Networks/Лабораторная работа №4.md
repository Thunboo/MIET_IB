# Настройка статических маршрутов, NAT, DHCP

## Задачи 
- Настройка статических маршрутов. 
- Определение различных типов маршрутов в таблице маршрутизации.
- Проверка конфигурации и возможности подключения.
- Настройка маршрутизатора в качестве сервера DHCP.
- Настройка клиентского маршрутизатора для использования NAT.
- Настройка безопасного соединения WEP между ПК и беспроводным маршрутизатором.
## Исходные данные 

У небольшой компании есть два офиса в разных зданиях. Оба офиса нуждаются в связи друг с другом и с веб-сервером, который размещается на мощностях поставщика услуг Интернета. На всех маршрутизаторах необходимо настроить статические маршруты для обмена данными.
У провайдера имеется несколько рабочих станций, которым необходимо автоматически настраивать IP-адреса в локальной подсети и настроить службу DHCP для выхода в Интернет. 
Поскольку для внутренней сети офиса настроен диапазон частных адресов, то для доступа в Интернет необходимо настроить статический NAT. 
Также в одном из офисов попросили установить беспроводной маршрутизатор WRT300N. Компания наняла новых работников, которые будут пользоваться беспроводными компьютерами, что позволит сэкономить деньги на добавлении дополнительных проводных соединений в здании. Руководителей данного предприятия волнует вопрос безопасности сети, так как по ней передаются финансовые и высоко секретные данные. Ваша задача заключается в настройке средств обеспечения безопасности на маршрутизаторе, чтобы защитить данные. 
Настройку всех устройств необходимо проводить из командной строки (за исключением беспроводного маршрутизатора).

## Задание 9: Настройка NAT Overload на маршрутизаторе Router1
### Router 1:
```
conf term

access-list 1 permit 192.168.4.0 0.0.0.255

ip nat inside source list 1 interface serial 0/0/1 overload

interface fastethernet 0/0
ip nat inside
exit

interface serial 0/0/0
ip nat inside
exit

interface serial 0/0/1
ip nat outside
exit

Ctrl+Z

copy run start
```

## Вопросы для самопроверки

- В чем разница между кодами C и S, указанными рядом с маршрутами в таблице маршрутизации?
  > 
  > **C (Connected)** — сеть напрямую подключена к маршрутизатору, появляется автоматически.
  > **S (Static)** — маршрут добавлен вручную командой `ip route`.
- Какая команда используется для создания статического маршрута к сети 192.168.10.0 255.255.255.0, если следующий переход - 172.16.200.1?
  `ip route 192.168.10.0 255.255.255.0 172.16.200.1`
- Для чего предназначен DHCP в клиентской сети?
  **DHCP** автоматически выдаёт:
  - IP-адрес
  - Маску подсети
  - Шлюз по умолчанию
  - DNS-сервер
- Какой IP-адрес назначается рабочей станции после обновления при включенном DHCP?
  > Рабочая станция получает **свободный IP-адрес из пула DHCP**, настроенного на сервере или маршрутизаторе.
- Какие еще параметры DHCP, не включенные в данное задание, можно задать на маршрутизаторе Cisco?
  - Доменное имя (`domain-name`)
  - Время аренды (`lease`)
  - NTP-сервер
  - TFTP-сервер
- В чем цель использования протокола WEP в беспроводной сети?
  > **WEP** используется для **шифрования** трафика в беспроводной сети, чтобы предотвратить **несанкционированный доступ**.
- В чем значение ключа, который используется для защиты протокола WEP?
  > Ключ **WEP** — это пароль, который используется для шифрования и расшифровки передаваемых данных.
- Является ли протокол WEP лучшим вариантом для защиты беспроводного доступа?
  > Нет. WEP считается устаревшим и небезопасным.  Лучше использовать **WPA2 или WPA3**.
- Каково назначение статического NAT?
  > **Статический NAT** создаёт постоянное сопоставление **одного внутреннего** IP-адреса **одному внешнему** IP-адресу.
- Какая команда используется для назначения внутреннего интерфейса статического NAT?
  ```
	interface fastethernet 0/0
	ip nat inside
  ```
- Каково назначение NAT Overload? Какие команды необходимо ввести для его включения?
  > **Назначение:**  Позволяет нескольким внутренним устройствам использовать один внешний IP-адрес за счёт разных портов.
  > **Команды:**
  
 ```
	access-list 1 permit 192.168.X.0 0.0.0.255  
	ip nat inside source list 1 interface fastethernet 0/1 overload  
	interface fastethernet 0/0  
	ip nat inside  
	interface fastethernet 0/1  
	ip nat outside
	```
