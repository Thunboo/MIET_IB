# Настройка, проверка и устранение неполадок VLAN

## Задачи 
- Провести правильную настройку домена, режима и пароля VTP.
- Выполнить настройку виртуальных сетей VLAN на сервере VTP и просмотреть изменения на клиенте VTP. 
- Провести наблюдение за конфигурацией VLAN коммутатора и его работой.
- Организовать связь между виртуальными сетями VLAN через подинтерфесы маршрутизатора. 
- Проверить конфигурацию VLAN и ее работу.
## Исходные данные 

Настройка домена VTP для корпоративной сети - это эффективный способ настройки и обслуживания VLAN в сети. Вы являетесь администратором сети, и вам необходимо создать рабочую среду VTP на одной из удаленных площадок. Это позволит вам и членам вашей группы более эффективно обслуживать сети VLAN из головного офиса.

## Вопросы для самопроверки

- Можно ли задать коммутатору IP-адрес?
  **Да**. На L2-коммутаторе IP обычно назначают **интерфейсу VLAN** для управления, а не физическому порту.
- Зачем нужна настройка VLAN в сети?
  Чтобы **логически разделить** одну физическую сеть на несколько отдельных широковещательных доменов. Это повышает **безопасность**, упрощает управление и уменьшает лишний broadcast-трафик.
- Что произойдет при добавлении нового коммутатора в существующую сеть, если настройка VTP как сервера выполнена корректно, а его номер версии конфигурации больше номеров остальных коммутаторов? 
  Его база VLAN может быть принята остальными коммутаторами как более новая. В результате VLAN на всей сети могут **перезаписаться**. Это одна из главных опасностей VTP.
- Почему восходящий канал для нового коммутатора должен быть настроен, как магистральный?
  Потому что **trunk** передает трафик **нескольких VLAN** и VTP-объявления между коммутаторами. Через access-порт VLAN между коммутаторами нормально не передаются.
- Что следует настроить для обеспечения связи между сетями VLAN? 
  Нужна **межвлановая маршрутизация**: роутер с подинтерфейсами, либо L3-коммутатор.
- Какой сети VLAN принадлежат все порты при отсутствии настройки конфигурации?
  По умолчанию все порты находятся в **VLAN 1**.
- Какие три команды необходимы для настройки и интеграции нового коммутатора в существующий домен VTP?
  ```
	vtp domain MIET  
	vtp mode client  
	vtp password ciscoX
  ```
- Какой режим VTP устанавливает по умолчанию коммутатор? 
  По умолчанию — **VTP Server**.
- Какие две команды следует использовать для удаления всех настроек VLAN и возврата к настройкам по умолчанию? 
  Обычно удаляют файл VLAN database и перезагружают устройство:
  ```
	delete flash:vlan.dat  
	reload
  ```
- Какую команду вы использовали бы для вывода данных о портах, назначенных каждой из VLAN?
  `show vlan brief`
- Что может препятствовать обновлениям VLAN на VTP-клиенте в одном домене с VTP-сервером?
	  - разные **VTP password**
	- разные **VTP version**
	- trunk не настроен или не работает
	- коммутаторы в разных **VTP domain**
	- VTP переведен в transparent
	- проблема с соединением
- Каким будет результат добавления в сеть нового коммутатора, настроенного в качестве VTP-сервера с номером версии конфигурации меньшим, чем у существующего VTP-сервера?
  Ничего критичного не произойдет: его база VLAN **не заменит** текущую, потому что номер ревизии меньше. Скорее он сам примет более новую конфигурацию от существующего VTP-сервера.
