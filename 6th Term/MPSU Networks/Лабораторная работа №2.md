## Задание 1: Разработка схемы подсетей

6 подсетей. 4<6<8 => Надо 3 бита (2^3 = 8). Тогда маска будет 24+3 = /27
11111111.11111111.11111111.11100000
255.255.255.(128+64+32) = 255.255.255.224

В подсети на узлы осталось 5 бит = 32 адреса. Значит для хостов: 32-2=30 адресов хостов

## Задание 4. Настройка клиентского коммутатора

- Подключитесь к консоли первого коммутатора с помощью консольного кабеля и программы эмуляции терминала на компьютере PC1.
- Настройте имя узла коммутатора. Задайте имя узла для первого коммутатора "<№варианта>Switch1".
- Настройте пароль и секретный пароль привилегированного режима. Все пароли должны соответствовать "cisco<№варианта>".
- Настройте пароль консоли. В режиме настройки линии задайте пароль "cisco<№варианта>" и укажите, что пароль нужно вводить при каждом входе в систему.
- Настройте пароль канала VTY от 0 до 15. В режиме настройки линии задайте пароль "cisco<№варианта>" и укажите, что пароль нужно вводить при каждом входе в систему.
- Настройте IP-адрес интерфейса VLAN1. Назначьте интерфейсу сети VLAN1 последний возможный IP-адрес и маску из подсети 0 (первой по счёту подсети).
- Настройте шлюз по умолчанию. Назначьте шлюзу по умолчанию первый возможный IP-адрес из подсети 0 (первой по счёту подсети).
- Убедитесь, что IP-адрес интерфейса управления коммутатора VLAN1 и IP-адрес узла PC1 находятся в одной локальной сети.
- Сохраните текущую конфигурацию в качестве начальной конфигурации коммутатора.
- Повторите пункты для соответствующего конфигурирования второго коммутатора.

```
enable
configure terminal

hostname 4Switch1

enable password cisco4
enable secret cisco4-123

line console 0
password cisco4
login
exit

line vty 0 15
password cisco4
login
exit

interface vlan 1
ip address 192.168.4.30 255.255.255.224
no shutdown
exit

# Сохраним конфиг
copy running-config startup-config
```

## Вопросы для самопроверки

- Помогает ли разделение на подсети уменьшить проблему нехватки IP-адресов? Поясните свой ответ.
  > Да. Потому что неминуемо у нас в топологии будут подсети, а если выдавать их с слишком большой маской - они быстро кончаться. Поэтому и бъем на более мелкие
- В схеме записей разработчика было указано, что в беспроводной подсети может быть до 30 ПК. Создает ли это ситуацию, когда IP-адреса могли бы использоваться непродуктивно? Важно ли это и почему?
  > У нас могут простаивать некоторые адреса = непродуктивно. В реальных сетях IPшники - ограниченный ресурс.
- Есть альтернативные методы разделения на подсети с использованием бесклассовой внутридоменной маршрутизации (CIDR) и сетевых масок переменной длины (VLSM). Явилось бы использование VLSM выгодным вариантом для разделения данной сети?
  > Да, так как тогда можно было бы выделить более мелкие маски на те подсети, где хостов заметно меньше, чем число IP адресов
- Почему нужно назначить IP-адрес именно интерфейсу VLAN1, а не просто любому интерфейсу FastEthernet?
  > Потому что на коммутаторе FastEthernet-порты работают на L2 и не имеют IP; IP нужен только интерфейсу управления VLAN, чтобы к коммутатору можно было обращаться по сети.
- Какая команда требуется для активизации идентификации с помощью пароля на консоли и в линиях VTY?
	```
	line console 0
	password <пароль>
	login

	line vty 0 15
	password <пароль>
	login
	```
